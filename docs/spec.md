# CTF Diary — 功能规格说明书

> 版本: v1.2 | 日期: 2026-06-03 | 状态: 已实现

---

## 1. 项目概述

**CTF Diary** 是一个桌面端 CTF 选手日常工具，集比赛管理、题目追踪、附件管理、日志打卡、笔记编辑、导出与通知于一体。界面美观，支持明暗主题和自定义背景图片。

## 2. 用户故事

### 2.1 比赛管理

**作为** CTF 选手
**我想要** 从 CTFtime 获取近期比赛信息
**以便** 规划我的参赛安排

验收标准:
- 展示 CTFtime 上近期比赛列表（名称、日期、权重、格式、链接）
- 提供"参加"按钮，点击后自动为比赛创建本地目录
- 区分"已参加"和"未参加"的比赛
- 支持查看比赛详情（时间线、进度、附件）

### 2.2 题目追踪

**作为** CTF 选手
**我想要** 跟踪每场比赛内各个题目的解决状态
**以便** 清晰掌握比赛完成进度

验收标准:
- 可为每个比赛创建题目（按分类：pwn/re/web/crypto/misc）
- 每个题目有独立文件夹存放附件和笔记
- 三态状态切换：未解决 → 尝试中 → 已解决 → 未解决
- 进度条可视化：比赛/分类级别的解决率
- 比赛全部题目解决后自动标记为"已解决"

### 2.3 附件管理

**作为** CTF 选手
**我想要** 为每场比赛管理附件文件
**以便** 整理和快速定位解题材料

验收标准:
- 每场比赛目录下自动创建 `pwn/` 和 `re/` 子目录
- 创建题目时自动在对应分类下创建题目文件夹
- 支持拖拽/导入附件文件
- 支持文件浏览、预览、删除
- 题目文件夹在文件树中显示解决状态图标
- 支持**导出比赛/题目文件**为 ZIP 压缩包

### 2.4 日志与打卡

**作为** CTF 选手
**我想要** 记录每天的学习内容和比赛进展
**以便** 追踪我的成长轨迹

验收标准:
- 日历视图，可视化打卡记录
- 每天可以写学习日志（支持 Markdown）
- 提供打卡（check-in）功能
- 统计连续打卡天数、总打卡数
- 日志支持标签分类（如: pwn, reverse, web, crypto, misc）

### 2.5 编辑器

**作为** CTF 选手
**我想要** 使用内置编辑器编写和运行代码
**以便** 高效完成解题和笔记

验收标准:
- 多标签页，同时编辑多个文件
- 左侧文件树，快速切换比赛文件
- Markdown 编辑器 + 实时渲染预览
- Python 编辑器 + 一键运行 + 输出面板
- 语法高亮（跟随主题明暗切换）
- 底部状态栏：文件类型、光标位置、保存状态
- 自动补全、括号匹配、行号、搜索（Ctrl+F）
- 编辑器填满整个可用区域

### 2.6 比赛通知

**作为** CTF 选手
**我想要** 收到比赛临近的系统通知
**以便** 不会错过即将开始的比赛

验收标准:
- 应用启动时扫描已参加的比赛
- 比赛开始前 N 天弹出系统原生通知
- 比赛已开始时弹出通知
- 同一比赛+日期不重复通知
- 点击通知自动恢复并聚焦 CTF Diary 窗口
- 可在设置中开启/关闭通知
- 可自由配置提前天数（1/2/3/5/7 天）

### 2.7 个性化设置

**作为** 用户
**我想要** 自定义工具的外观和存储位置
**以便** 让工具更符合我的审美和习惯

验收标准:
- 自定义背景图片（支持本地图片上传 + 清除）
- 明暗主题切换
- 卡片透明度可调
- 字体大小调整
- 界面语言切换（中/英）
- 自定义比赛文件存储路径
- 通知开关 + 提前天数设置
- 数据管理（统计概览 + 分项清理 + 重置）

## 4. 技术约束

- 桌面应用，支持 Windows 平台
- 本地优先，数据存储于本地文件系统 + SQLite
- 界面响应式、现代化
- 需访问 CTFtime API (https://ctftime.org/api/v1/)

## 5. 非功能需求

- 启动速度 < 3 秒
- 界面流畅无卡顿
- 数据本地存储，无隐私泄露
- 离线可用（比赛信息支持缓存）
- 自定义无边框窗口，融入整体设计
- 编辑器和预览面板均跟随明暗主题
- 编辑器填满整个可用视图区域
- 导出为标准 ZIP 格式，通用性强
